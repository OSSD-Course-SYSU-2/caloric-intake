🔥 LightCalorie 轻燃热量记录 APP
HarmonyOS 原生减脂热量管理工具
LightCalorie 是一款面向健身、减脂、塑形人群的鸿蒙原生轻量化热量管理应用。项目基于 ArkTS + ArkUI 开发，遵循 Stage 模型规范，采用纯本地离线架构，无需云端服务，实现饮食热量摄入、运动热量消耗全流程记录，搭配智能热量计算、数据可视化、体重追踪与目标提醒能力，帮助用户科学管理热量缺口，高效完成减脂健身目标。
应用适配鸿蒙多设备形态，UI 自适应手机、平板、折叠屏设备，轻量化低功耗，数据本地持久化留存，上手零学习成本。

✨ 功能特性
🎯 核心热量管理
- 个性化热量测算：录入身高、体重、年龄、性别、运动习惯与健身目标，自动计算 BMR 基础代谢、TDEE 每日总消耗、推荐摄入热量与减脂热量缺口
- 饮食热量摄入记录：分早餐/午餐/晚餐/加餐四时段记录，内置海量通用食物热量库，支持自定义食物录入与份数调整，实时统计当日摄入总额
- 运动热量消耗记录：覆盖跑步、跳绳、力量训练、骑行等主流运动，根据体重、时长、强度自动核算消耗，支持自定义运动类型
- 热量缺口统计：实时计算每日收支差额，直观展示减脂有效缺口/热量盈余，科学指导减脂
📊 数据可视化与进度追踪
- 日维度数据统计：实时展示总摄入、总消耗、剩余可摄入额度、热量缺口核心数据
- 周期趋势图表：支持 7 日/30 日热量收支折线图、柱状图可视化展示
- 体重进度追踪：每日体重录入，生成体重变化曲线，关联热量数据分析减脂效果
- 历史数据回溯：按日期精准查询过往饮食、运动、热量全部记录数据
🔔 目标打卡与智能提醒
- 个性化目标设置：自定义每日热量上限、每日运动时长、每周减脂目标
- 超标智能提醒：热量摄入超标时弹窗+状态栏双重提示，及时纠正饮食偏差
- 定时打卡提醒：自定义饮食记录、运动打卡提醒时间，养成自律习惯
- 目标完成标识：每日达标后展示专属打卡成果，可视化激励用户坚持减脂
📱 多设备自适应适配
- 多尺寸断点适配：适配手机、平板、折叠屏常规尺寸断点
- 折叠屏动态适配：监听折叠状态变化，自动调整页面布局、卡片尺寸、字体大小
- 全局响应式 UI：页面组件、数据卡片、图表尺寸随设备分辨率动态自适应
💾 本地数据持久化管理
- 基础信息永久留存：用户身体数据、热量参数本地持久化，无需重复录入
- 记录数据永久保存：所有饮食、运动、体重历史记录本地留存，零点仅重置当日统计，不清除历史数据
- 自定义数据存储：用户新增的自定义食物、自定义运动数据本地持久化
- 自主数据管理：支持单条/全部记录删除、本地数据导出
🌓 个性化界面体验
- 深浅主题切换：支持亮色/暗色模式一键切换，适配系统显示偏好
- 字体自适应：全局字体大小适配设备，提升阅读体验
- 轻量化低功耗：无后台冗余进程，启动快速，操作流畅无卡顿

🧠 核心业务逻辑
热量计算核心公式
- 热量缺口公式：每日热量缺口 = 每日总消耗(TDEE) − 每日总摄入热量
- 减脂逻辑：缺口为正 = 有效减脂；缺口为负 = 热量盈余，易堆积脂肪
- BMR基础代谢：采用行业通用男女差异化计算公式，精准匹配用户身体数据
- TDEE总消耗：基础代谢 × 日常活动系数，匹配久坐/轻度/中度/重度运动场景
数据更新机制
- 每日零点自动重置当日统计数据，历史所有记录永久本地留存
- 新增/编辑/删除饮食、运动记录时，实时刷新全局热量统计数据
- 用户个人信息修改后，自动重新计算全部热量参数并同步更新
- 系统默认食物、运动热量参数支持用户自定义覆盖，适配个性化饮食运动场景
🏗️ 架构设计
状态管理机制
采用 页面视图 + 数据模型分层解耦 架构，独立 Model 层管理核心业务数据，页面通过状态监听实现视图自动刷新，规避ArkTS嵌套对象响应式失效问题。
- 视图层：Page 页面、自定义组件，仅负责UI展示与用户交互
- 状态层：@State / @Prop 管理页面临时状态，监听数据模型变更
- 模型层：UserModel / FoodModel / SportModel 统一管理业务数据、计算逻辑、存储读写
- 工具层：独立封装计算、存储、适配工具类，复用全局逻辑
